🌙 DAILY — Recovery card

Content

  • Recovery advice driven by drink type (tequila ≠ beer ≠ wine ≠ spirits)
  • If good night → praise it, reinforce what worked
  • If heavy night → recovery tips + suggest a smarter option for next time
  • No stats recap. No "you should drink less."

Voice

Friend who knows what tequila does to you. Short, practical, tonight-actionable.

UI changes

  • Sections always open (no collapse)
  • Colored pill labels instead of left border bars
  • Keep: header, serif body text, EKG animation while loading
📅 WEEKLY — Story card

Content (3 sections)

  • The week — what happened, named the outlier night
  • What you nailed — code finds one thing they actually did well, AI calls it out
  • Next week — one specific fix or rhythm to try (e.g. one tough night, one chill night)

Voice

Casual, forward-looking. 3 sentences max per section. No stats recapping.

UI changes

Same as daily — pill labels, open sections, keep header + serif + EKG.

🗓️ MONTHLY — Deep analysis card

Content (4 sections — unchanged structure)

  • Medical Analysis — clinical, honest, body-focused
  • Nutrition & Metabolism — drink choices, caloric impact, one tip
  • Behavioral Insight — the pattern they keep repeating (signature move)
  • Overall Synthesis — two sentences tying it together

Voice

Clinical and direct — the monthly earns the formality. No moralizing, but honest.

UI changes

Pill labels replace left-border bars. Sections always open. Keep header + serif + EKG.

⚙️ Implementation approach — C
  • iOS / Firebase pre-computes insights in code (best night, outlier night, signature habit, drink type breakdown)
  • Those facts get passed to Claude as structured data
  • Claude writes the narrative — it doesn't have to figure anything out
  • All prompt changes in functions/index.js
  • UI changes in CoachReportCard.swift and AIReportCard.swift
